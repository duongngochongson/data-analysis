Query 08 · Cohort funnel: view → cart → purchase (Q1 2017)

WITH productview AS (
    SELECT FORMAT_DATE("%Y%m", PARSE_DATE("%Y%m%d", date)) AS month,
        COUNT(eCommerceAction.action_type) AS num_product_view
    FROM `bigquery-public-data.google_analytics_sample.ga_sessions_2017*`, UNNEST(hits) AS hits
    WHERE _table_suffix BETWEEN '0101' AND '0331'
        AND eCommerceAction.action_type = '2'
    GROUP BY month
),
cart AS (
    SELECT FORMAT_DATE("%Y%m", PARSE_DATE("%Y%m%d", date)) AS month,
        COUNT(eCommerceAction.action_type) AS num_addtocart
    FROM `bigquery-public-data.google_analytics_sample.ga_sessions_2017*`, UNNEST(hits) AS hits
    WHERE _table_suffix BETWEEN '0101' AND '0331'
        AND eCommerceAction.action_type = '3'
    GROUP BY month
),
purchase AS (
    SELECT FORMAT_DATE("%Y%m", PARSE_DATE("%Y%m%d", date)) AS month,
        COUNT(eCommerceAction.action_type) AS num_purchase
    FROM `bigquery-public-data.google_analytics_sample.ga_sessions_2017*`,
        UNNEST(hits) AS hits, UNNEST(hits.product) AS product
    WHERE _table_suffix BETWEEN '0101' AND '0331'
        AND eCommerceAction.action_type = '6'
        AND product.productRevenue IS NOT NULL
        AND totals.transactions IS NOT NULL
    GROUP BY month
)
SELECT month, num_product_view, num_addtocart, num_purchase,
    ROUND(num_addtocart / num_product_view * 100, 2) AS add_to_cart_rate,
    ROUND(num_purchase  / num_product_view * 100, 2) AS purchase_rate
FROM productview JOIN cart USING (month) JOIN purchase USING (month)
ORDER BY month;

Python RFM Model Analysis

Segment SuperStore customers by Recency, Frequency, and Monetary value to power targeted marketing campaigns.

Platform: Google Colab. Library: Pandas, NumPy, Seaborn, Matplotlib, Squarify. Technique: EDA, data transformations, aggregation and grouping.

Context: SuperStore, a global retail company, seeks to segment its large customer base for marketing campaigns. The RFM model scores each customer on three dimensions: how recently they bought, how often, and how much they spent.

1 · Data Cleaning

The dataset has two tables: transaction information (many-to-one: each order ID → multiple product IDs) and a segmentation lookup table.

Import libraries

import pandas as pd
import numpy as np
import matplotlib.pyplot as plt

Import data

data_link = '1yNt8-kkoDyYzq8tYbqWRqqrAfyhPNtBlfSo-9aRvbCY'
data = gc.open_by_key(data_link)

order_info   = pd.DataFrame(data.worksheet('ecommerce retail').get_all_records())
segmentation = pd.DataFrame(data.worksheet('Segmentation').get_all_records())

Prepare segmentation lookup

segmentation.columns = ['segment', 'score']
segmentation['score'] = segmentation['score'].str.split(',')
segmentation = segmentation.explode('score')
segmentation['score'] = segmentation['score'].str.strip()

Clean order info

# Drop duplicates
order_info = order_info.drop_duplicates()

# Drop cancelled orders (InvoiceNo starts with 'C')
order_info = order_info[~order_info['InvoiceNo'].astype(str).str.startswith('C')]

# Remove bad-debt rows where InvoiceNo is not int
order_info = order_info[order_info['InvoiceNo'].apply(lambda x: isinstance(x, int))]

# Keep only valid rows
order_info = order_info[
    (order_info['Quantity'] > 0) &
    (order_info['CustomerID'] != '') &
    (order_info['UnitPrice'] > 0)
]

# Parse dates
order_info['InvoiceDate'] = pd.to_datetime(order_info['InvoiceDate'],
                                            format='%m/%d/%y %H:%M', errors='coerce')
order_info['Date'] = order_info['InvoiceDate'].dt.date.astype('datetime64[ns]')
order_info = order_info.drop(columns=['InvoiceDate'])

# Rename columns
order_info = order_info.rename(columns={
    'InvoiceNo': 'invoiceid', 'StockCode': 'stockcode',
    'Description': 'description', 'Quantity': 'quantity',
    'UnitPrice': 'price', 'CustomerID': 'customerid',
    'Country': 'country', 'Date': 'date'
})

2 · RFM Segmentation

# Revenue column
order_info['revenue'] = order_info['quantity'] * order_info['price']

# Aggregate per customer
customer_rfm = order_info.groupby('customerid').agg(
    {'date': 'max', 'invoiceid': 'nunique', 'revenue': 'sum'}
).reset_index()

# Recency = days from 31/12/2011 to last purchase
customer_rfm['date'] = (
    pd.to_datetime('31/12/2011', format='%d/%m/%Y') - customer_rfm['date']
).dt.days
customer_rfm.columns = ['customerid', 'recency', 'frequency', 'monetory']

# Score 1-5 with quintiles (R inverted: lower recency = higher score)
customer_rfm['r_rank'] = 6 - (pd.qcut(customer_rfm['recency'].rank(method='first'), 5, labels=False) + 1)
customer_rfm['f_rank'] = pd.qcut(customer_rfm['frequency'].rank(method='first'), 5, labels=False) + 1
customer_rfm['m_rank'] = pd.qcut(customer_rfm['monetory'].rank(method='first'),  5, labels=False) + 1

# 3-digit score string, merge with segment lookup
customer_rfm['score'] = customer_rfm['r_rank'].astype(str) + customer_rfm['f_rank'].astype(str) + customer_rfm['m_rank'].astype(str)
customer_rfm = segmentation.merge(customer_rfm, on='score', how='right')

# Aggregate by segment
rfm = customer_rfm.groupby('segment').agg(
    num_ctm=('customerid', 'count'),
    avg_r=('recency',  'mean'),
    avg_f=('frequency','mean'),
    sum_rvn=('monetory','sum')
).round(1).reset_index()

4 · Insights & Recommendations

By end of 2011, SuperStore had a mixed business situation: strong segments (Champions, Loyal, Potential Loyalist) alongside weak ones (Hibernating, About to Sleep, At Risk, Lost). Champions made up 19.2% of total customers; Hibernating 15.9%.

Clear skewed averages indicate a significant gap between strong and weak segments. Heavy reliance on Champions suggests issues in turning new customers into loyal ones. Prioritize Recency and Frequency over Monetary — loyal shoppers make many purchases at lower amounts throughout the year.

Customer Churn Prediction & Segmentation

Combine RFM segmentation with churn signals to target retention campaigns at the right customers.

Platform: Google Colab. Library: Pandas, NumPy, Seaborn, Matplotlib, Squarify; scikit-learn for the modeling step. Technique: EDA, data transformations, aggregation; logistic regression and gradient boosting for churn prediction.

Context: SuperStore wants to retain loyal customers and re-engage those at risk. Marketing needs a per-customer retention priority score, not just a broad segment label.

3 · From Segmentation to Churn Prediction

The next step adds a binary churn label (churned = no purchase in N days) and engineers features: R, F, M values, customer tenure, category mix, and average order gap.
