SQL · BigQuery

SQL eCommerce User Behavior Exploration

Analyzing user engagement and revenue trends on the Google Analytics public dataset in BigQuery.

Main Techniques: Window functions, CTEs, UNNEST on nested hits and product arrays.

Query 01 · Total visits, pageviews, transactions (Jan–Mar 2017)

SELECT
  FORMAT_DATE("%b-%Y", PARSE_DATE("%Y%m%d", date)) AS month,
  SUM(totals.visits)       AS number_of_visit,
  SUM(totals.pageviews)    AS number_of_pageview,
  SUM(totals.transactions) AS number_of_transaction
FROM `bigquery-public-data.google_analytics_sample.ga_sessions_2017*`
WHERE _table_suffix BETWEEN '0101' AND '0331'
GROUP BY month;

March 2017 had the most transactions (993); January had the highest visits (64,694).

Query 02 · Bounce rate per traffic source (Jul 2017)

SELECT
  trafficSource.source,
  COUNT(visitNumber)                                        AS total_visits,
  SUM(totals.bounces)                                       AS total_bounces,
  ROUND(SUM(totals.bounces)/COUNT(visitNumber)*100, 2)      AS bounce_rate
FROM `bigquery-public-data.google_analytics_sample.ga_sessions_201707*`
GROUP BY trafficSource.source
ORDER BY total_visits DESC;

Google led in visits (38,400). YouTube.com and m.facebook.com had the highest bounce rates at 66.73% and 64.28%.

Query 03 · Revenue by traffic source, weekly & monthly (Jun 2017)

WITH monthly_revenue AS (
  SELECT "month" AS time_type,
    FORMAT_DATE("%m-%Y", PARSE_DATE("%Y%m%d", date)) AS time,
    trafficSource.source AS source,
    ROUND(SUM(product.productRevenue/1000000), 2)    AS revenue
  FROM `bigquery-public-data.google_analytics_sample.ga_sessions_201706*`,
    UNNEST(hits) AS hits, UNNEST(hits.product) AS product
  WHERE product.productRevenue IS NOT NULL
  GROUP BY time_type, time, source
),
weekly_revenue AS (
  SELECT "week" AS time_type,
    FORMAT_DATE("%W-%Y", PARSE_DATE("%Y%m%d", date)) AS time,
    trafficSource.source AS source,
    ROUND(SUM(product.productRevenue)/1000000, 2)    AS revenue
  FROM `bigquery-public-data.google_analytics_sample.ga_sessions_201706*`,
    UNNEST(hits) AS hits, UNNEST(hits.product) AS product
  WHERE product.productRevenue IS NOT NULL
  GROUP BY time_type, time, source
)
SELECT * FROM monthly_revenue
UNION ALL SELECT * FROM weekly_revenue
ORDER BY revenue DESC;

Direct traffic dominated June 2017 with ~$97k monthly revenue, far ahead of Google ($18.7k) and DFA ($8.8k).

Query 04 · Avg pageviews: purchasers vs non-purchasers (Jun–Jul 2017)

WITH purchase_data AS (
  SELECT FORMAT_DATE("%m-%Y", PARSE_DATE("%Y%m%d", date)) AS month,
    ROUND(SUM(totals.pageviews)/COUNT(DISTINCT fullVisitorId), 2) AS avg_pageviews_purchase
  FROM `bigquery-public-data.google_analytics_sample.ga_sessions_2017*`,
    UNNEST(hits) AS hits, UNNEST(hits.product) AS product
  WHERE totals.transactions IS NOT NULL AND product.productRevenue IS NOT NULL
    AND _table_suffix BETWEEN '0601' AND '0731'
  GROUP BY month
),
non_purchase_data AS (
  SELECT FORMAT_DATE("%m-%Y", PARSE_DATE("%Y%m%d", date)) AS month,
    ROUND(SUM(totals.pageviews)/COUNT(DISTINCT fullVisitorId), 2) AS avg_pageviews_non_purchase
  FROM `bigquery-public-data.google_analytics_sample.ga_sessions_2017*`,
    UNNEST(hits) AS hits, UNNEST(hits.product) AS product
  WHERE totals.transactions IS NULL AND product.productRevenue IS NULL
    AND _table_suffix BETWEEN '0601' AND '0731'
  GROUP BY month
)
SELECT p.month, p.avg_pageviews_purchase, n.avg_pageviews_non_purchase
FROM purchase_data p
FULL OUTER JOIN non_purchase_data n USING (month)
ORDER BY p.month;

Purchasers viewed significantly more pages than non-purchasers in both months; July had slightly higher averages than June.

Query 05 · Avg transactions per purchasing user (Jul 2017)

SELECT
  "201707" AS month,
  ROUND(SUM(totals.transactions)/COUNT(DISTINCT fullVisitorId), 2)
    AS avg_total_transactions_per_user
FROM `bigquery-public-data.google_analytics_sample.ga_sessions_201707*`,
  UNNEST(hits) AS hits, UNNEST(hits.product) AS product
WHERE totals.transactions >= 1 AND product.productRevenue IS NOT NULL;

Purchasing users averaged 4.16 transactions in July 2017.

Query 06 · Avg revenue per session for purchasers (Jul 2017)

SELECT
  FORMAT_DATE('%Y-%m', PARSE_DATE('%Y%m%d', date)) AS month,
  ROUND((SUM(product.productRevenue)/SUM(totals.visits))/1000000, 2)
    AS avg_revenue_by_user_per_visit
FROM `bigquery-public-data.google_analytics_sample.ga_sessions_201707*`,
  UNNEST(hits) AS hits, UNNEST(hits.product) AS product
WHERE product.productRevenue IS NOT NULL AND totals.transactions IS NOT NULL
GROUP BY month;

Average revenue per session reached $43.86 in July 2017.

Query 07 · Cross-sell from "YouTube Men's Vintage Henley" (Jul 2017)

WITH henley_buyers AS (
  SELECT DISTINCT fullVisitorId
  FROM `bigquery-public-data.google_analytics_sample.ga_sessions_201707*`,
    UNNEST(hits) AS hits, UNNEST(hits.product) AS product
  WHERE product.v2ProductName = "YouTube Men's Vintage Henley"
    AND totals.transactions >= 1 AND product.productRevenue IS NOT NULL
)
SELECT
  product.v2ProductName AS other_purchased_products,
  SUM(product.productQuantity) AS quantity
FROM `bigquery-public-data.google_analytics_sample.ga_sessions_201707*`,
  UNNEST(hits) AS hits, UNNEST(hits.product) AS product
WHERE fullVisitorId IN (SELECT * FROM henley_buyers)
  AND product.v2ProductName <> "YouTube Men's Vintage Henley"
  AND totals.transactions >= 1 AND product.productRevenue IS NOT NULL
GROUP BY product.v2ProductName
ORDER BY quantity DESC;

Top cross-sells: Google Sunglasses (20 units), Women's Vintage Hero Tee (7), SPF-15 Lip Balm (6) — strong affinity for Google/YouTube branded casual wear.

Query 08 · Cohort funnel: view → cart → purchase (Q1 2017)

WITH productview AS (
  SELECT FORMAT_DATE("%Y%m", PARSE_DATE("%Y%m%d", date)) AS month,
    COUNT(eCommerceAction.action_type) AS num_product_view
  FROM `bigquery-public-data.google_analytics_sample.ga_sessions_2017*`, UNNEST(hits) AS hits
  WHERE _table_suffix BETWEEN '0101' AND '0331' AND eCommerceAction.action_type = '2'
  GROUP BY month
),
cart AS (
  SELECT FORMAT_DATE("%Y%m", PARSE_DATE("%Y%m%d", date)) AS month,
    COUNT(eCommerceAction.action_type) AS num_addtocart
  FROM `bigquery-public-data.google_analytics_sample.ga_sessions_2017*`, UNNEST(hits) AS hits
  WHERE _table_suffix BETWEEN '0101' AND '0331' AND eCommerceAction.action_type = '3'
  GROUP BY month
),
purchase AS (
  SELECT FORMAT_DATE("%Y%m", PARSE_DATE("%Y%m%d", date)) AS month,
    COUNT(eCommerceAction.action_type) AS num_purchase
  FROM `bigquery-public-data.google_analytics_sample.ga_sessions_2017*`,
    UNNEST(hits) AS hits, UNNEST(hits.product) AS product
  WHERE _table_suffix BETWEEN '0101' AND '0331'
    AND eCommerceAction.action_type = '6'
    AND product.productRevenue IS NOT NULL AND totals.transactions IS NOT NULL
  GROUP BY month
)
SELECT month, num_product_view, num_addtocart, num_purchase,
  ROUND(num_addtocart/num_product_view*100, 2) AS add_to_cart_rate,
  ROUND(num_purchase/num_product_view*100,  2) AS purchase_rate
FROM productview JOIN cart USING (month) JOIN purchase USING (month)
ORDER BY month;

Retailer RFM Segmentation

Segment SuperStore customers by Recency, Frequency, and Monetary value to power targeted marketing campaigns.

Platform: Google Colab. Library: Pandas, NumPy, Seaborn, Matplotlib, Squarify.

Context: SuperStore, a global retailer, needs to segment its large customer base for campaign personalization. The RFM model scores each customer on three dimensions: how recently they bought, how often, and how much they spent.

Data Cleaning

  • Drop duplicate rows.
  • Remove cancelled orders (InvoiceNo starting with C).
  • Drop bad-debt adjustment rows where InvoiceNo is not an integer.
  • Keep only rows with Quantity > 0, UnitPrice > 0, and non-empty CustomerID.
  • Parse InvoiceDate to datetime, extract Date column.

RFM Pipeline

  1. Compute revenue = quantity × price.
  2. Aggregate per customer: last purchase date, unique invoice count, total revenue.
  3. Recency = days from 31/12/2011 to last purchase.
  4. Score R, F, M each 1–5 with pd.qcut quintiles (R inverted: lower recency = higher score).
  5. Concatenate to a 3-digit score, merge with segmentation lookup table.
  6. Aggregate segments: customer count, avg R, avg F, total revenue.

Visualization

Insights & Recommendations

Champions (19.2%) and Hibernating (15.9%) are the two largest groups — a wide gap between the best and most-disengaged customers. SuperStore over-relies on Champions, suggesting friction in converting new buyers to loyal ones. Prioritize Recency and Frequency over Monetary: loyal customers tend to buy often at smaller amounts.

Purchasing Department Performance

Operational dashboards for the AdventureWorks purchasing team — vendor reliability and cost optimization.

Approach: Apply design thinking to identify key metrics, pull data from BigQuery, transform in Power Query, model relationships, and build DAX measures.

Context: AdventureWorks is a fictitious bicycle manufacturer. The purchasing department must supply enough goods on time at optimal import prices.

Design Thinking · 5W1H

WhoPurchasing Manager, accountant.
Key stakeholderPurchasing Manager.
ProblemEnhance vendor oversight, promote data-driven decisions, optimize vendor relationships and costs.

WhyUnderstand vendor performance and evaluate procurement decisions.

North Star Metric

% No-error Orders — an order is error-free when received quantity equals stockable quantity. This maps directly to the team’s goal: keep production supplied without shortfalls or waste.

Ideate · Metric Layers

Layer 0 (totals): % no-error orders, % received, % rejected, % extra fee.
Layer 1 (breakdowns): over time, by ship method, by sub-category.

Machine Learning · Python

Customer Churn Prediction & Segmentation

Combine RFM segmentation with churn signals to target retention campaigns at the right customers.

Platform: Google Colab. Library: Pandas, NumPy, Seaborn, Matplotlib, Squarify; scikit-learn for the modeling step.

Context: SuperStore wants to retain loyal customers and re-engage those at risk. Marketing needs a per-customer retention priority score, not a broad segment label.

Segmentation Foundation (RFM)

  1. Clean transactions: drop cancelled orders, negative quantities, missing CustomerID.
  2. Compute Recency, Frequency, Monetary per customer.
  3. Score each dimension 1–5 with pd.qcut quintiles; build a 3-digit RFM score string.
  4. Map scores to seven segments via lookup table: Champions, Loyal, Potential Loyalist, Hibernating, About to Sleep, At Risk, Lost.

From Segmentation to Churn Prediction

The planned next step adds a binary label (churned = no purchase in N days) and engineers features: R, F, M values, customer tenure, category mix, and average order gap. A logistic regression baseline is trained first, then gradient boosting for lift. Output: churn probability per customer, combined with segment label to rank retention priority.

Visualization (Segmentation)

Why It Matters

About-to-Sleep and At-Risk together represent a sizeable share of the customer base. Knowing who is actually at risk — not just which segment they belong to — lets marketing concentrate budget on customers most likely to respond, rather than spraying discounts across an entire label.
